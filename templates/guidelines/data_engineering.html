Programming Fundamentals

Core Languages

  • Python: Most widely used language in data engineering
  • Java: Common for enterprise data engineering applications
  • Scala: Functional programming language for Apache Spark
  • SQL: Essential for database operations and transformations

Python Libraries

  • Pandas: Data manipulation and analysis
  • NumPy: Numerical computing
  • PySpark: Python API for Apache Spark
  • SQLAlchemy: SQL toolkit and ORM
  • FastAPI/Flask: API development

Software Engineering Practices

  • Version Control: Git, GitHub, GitLab
  • Testing: Unit testing, integration testing, pytest
  • Documentation: Docstrings, README files, Sphinx
  • CI/CD: Continuous integration and deployment
  • Design Patterns: Software architecture for data systems
Database Systems

Relational Databases

  • PostgreSQL: Advanced open-source RDBMS
  • MySQL/MariaDB: Popular open-source databases
  • SQL Server: Microsoft's enterprise database
  • Oracle: Enterprise-grade RDBMS

NoSQL Databases

  • MongoDB: Document store
  • Cassandra: Wide-column store for high scalability
  • Redis: In-memory key-value store
  • Neo4j: Graph database
  • Elasticsearch: Full-text search engine and document store

Data Warehouses

  • Snowflake: Cloud data warehouse
  • Amazon Redshift: AWS data warehouse
  • Google BigQuery: Google Cloud's serverless data warehouse
  • Azure Synapse: Microsoft's analytics service

Data Modeling

  • ER Modeling: Entity-relationship diagrams
  • Star Schema: Dimensional modeling for analytics
  • Snowflake Schema: Extended star schema
  • Data Vault: Hybrid approach for enterprise data warehousing
  • Kimball vs. Inmon: Data warehouse methodologies
Data Processing

Batch Processing

  • Apache Hadoop: Distributed storage and processing
  • Apache Spark: In-memory data processing engine
  • Hive: Data warehouse software for Hadoop
  • Pig: High-level platform for Hadoop
  • MapReduce: Programming model for processing large datasets

Stream Processing

  • Apache Kafka: Distributed event streaming platform
  • Apache Flink: Stream processing framework
  • Spark Streaming: Extension of Spark for streaming
  • Apache Pulsar: Cloud-native distributed messaging
  • AWS Kinesis: Real-time streaming data service

Data Transformation

  • ETL vs ELT: Extract, Transform, Load approaches
  • dbt: Data build tool for transformation in warehouses
  • Spark SQL: SQL interface for Spark
  • Data Quality: Validation, cleaning, standardization
  • Change Data Capture (CDC): Tracking data changes
Data Infrastructure

Cloud Platforms

  • AWS: Amazon's cloud computing platform
  • Google Cloud Platform (GCP): Google's cloud services
  • Microsoft Azure: Microsoft's cloud computing service
  • Cloud Data Services: S3, GCS, Blob Storage, etc.

Containerization & Orchestration

  • Docker: Containerization platform
  • Kubernetes: Container orchestration
  • Docker Compose: Multi-container applications
  • Helm: Kubernetes package manager

Infrastructure as Code

  • Terraform: Infrastructure provisioning
  • CloudFormation: AWS infrastructure as code
  • Ansible: Configuration management
  • Pulumi: Infrastructure as actual code

Monitoring & Logging

  • Prometheus: Monitoring and alerting
  • Grafana: Visualization and dashboards
  • ELK Stack: Elasticsearch, Logstash, Kibana
  • Datadog: Monitoring as a service
Data Pipelines & Orchestration

Orchestration Tools

  • Apache Airflow: Workflow management platform
  • Luigi: Python package for building pipelines
  • Dagster: Data orchestrator for ML, analytics, and ETL
  • Prefect: Modern workflow management
  • AWS Step Functions: Serverless workflow service

ETL/ELT Tools

  • Apache NiFi: Data ingestion and transformation
  • Informatica: Enterprise data integration
  • Talend: Data integration platform
  • Fivetran: Automated data integration
  • Stitch: Cloud ETL service

Data Quality & Governance

  • Great Expectations: Data validation
  • Apache Atlas: Data governance and metadata management
  • Collibra: Enterprise data governance
  • Alation: Data catalog
  • dbt Tests: Testing in transformation workflows

Pipeline Design Patterns

  • Idempotent Pipelines: Safe for multiple executions
  • End-to-End Testing: Testing entire pipelines
  • Backfilling: Reprocessing historical data
  • Incremental Processing: Handling only new/changed data
  • Error Handling: Managing pipeline failures
